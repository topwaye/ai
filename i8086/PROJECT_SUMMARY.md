# Intel 8086 操作系统项目总结

## 项目概述

本项目实现了一个基于 Intel 8086 CPU 的简单操作系统，包含自定义硬件设备驱动和中断处理功能。操作系统支持通过 0x60 中断（96号中断）接收自定义硬件设备的通知。

## 核心组件

### 1. 引导加载程序 (boot.asm)
**功能**：
- 从软盘启动系统
- 加载内核到内存地址 0x8000
- 显示启动信息

**关键代码位置**：
- 启动入口：`boot.asm:3`
- 内核加载：`boot.asm:load_kernel`
- 内核跳转：`boot.asm:20`

**内存布局**：
- 引导程序：0x7C00
- 内核：0x8000

### 2. 操作系统内核 (kernel.asm)
**功能**：
- 初始化系统环境（段寄存器、栈）
- 设置中断向量表（IVT）
- 初始化设备驱动
- 主事件循环

**关键代码位置**：
- 内核入口：`kernel.asm:3`
- IVT 设置：`kernel.asm:setup_ivt`
- 设备初始化：`kernel.asm:init_device_driver`
- 主循环：`kernel.asm:main_loop`

**中断向量表设置**：
- 0x60 中断向量地址：0x0000:0x0180
- 中断处理程序：`interrupts.asm:int60_handler`

### 3. 中断处理程序 (interrupts.asm)
**功能**：
- 处理 0x60 中断（自定义硬件设备中断）
- 保存和恢复寄存器上下文
- 区分读/写操作完成中断
- 发送 EOI（End of Interrupt）信号

**关键代码位置**：
- 中断入口：`interrupts.asm:int60_handler`
- 读完成处理：`interrupts.asm:handle_read_complete`
- 写完成处理：`interrupts.asm:handle_write_complete`

**中断处理流程**：
1. 保存寄存器
2. 读取设备状态（端口 0x60）
3. 判断中断类型
4. 执行相应处理
5. 发送 EOI
6. 恢复寄存器
7. 中断返回

### 4. 设备驱动 (device_driver.asm)
**功能**：
- 设备初始化和复位
- 设备读操作
- 设备写操作
- 设备状态查询
- 错误处理

**关键代码位置**：
- 设备初始化：`device_driver.asm:device_init`
- 读操作：`device_driver.asm:device_read`
- 写操作：`device_driver.asm:device_write`
- 状态查询：`device_driver.asm:device_get_status`

**设备接口规范**：

| 端口地址 | 功能 | 方向 |
|---------|------|------|
| 0x60 | 状态寄存器 | 输入 |
| 0x61 | 数据端口 | 双向 |
| 0x62 | 控制寄存器 | 输出 |

**状态寄存器位定义**：
- bit 0: 读数据就绪
- bit 1: 写数据就绪
- bit 6: 错误标志
- bit 7: 设备忙标志

**控制命令**：
- 0x01: 读命令
- 0x02: 写命令
- 0xFF: 复位命令

### 5. 测试程序 (test.asm)
**功能**：
- 测试设备驱动功能
- 演示读写操作
- 显示测试结果

**测试序列**：
1. 读取设备状态
2. 写入数据到设备
3. 从设备读取数据
4. 显示结果

## 中断处理机制

### 中断向量表（IVT）
- 位置：内存地址 0x0000
- 大小：1024 字节（256 个中断向量 × 4 字节）
- 每个中断向量格式：
  - 偏移地址（2 字节）
  - 段地址（2 字节）

### 0x60 中断处理流程

```
硬件设备触发中断
    ↓
CPU 识别中断号 (0x60)
    ↓
CPU 从 IVT 读取中断向量 (0x0000:0x0180)
    ↓
CPU 保存上下文（标志寄存器、CS、IP）
    ↓
跳转到中断处理程序 (int60_handler)
    ↓
保存通用寄存器
    ↓
读取设备状态寄存器
    ↓
判断中断类型（读/写/错误）
    ↓
执行相应处理
    ↓
发送 EOI 到 8259A
    ↓
恢复寄存器
    ↓
IRET（中断返回）
    ↓
CPU 恢复上下文，继续执行
```

## 系统启动流程

```
BIOS 加载引导程序到 0x7C00
    ↓
引导程序执行 (boot.asm:start)
    ↓
初始化段寄存器和栈
    ↓
显示启动消息
    ↓
加载内核到 0x8000
    ↓
跳转到内核 (0x0000:0x8000)
    ↓
内核执行 (kernel.asm:start)
    ↓
设置中断向量表
    ↓
初始化设备驱动
    ↓
启用中断 (STI)
    ↓
显示欢迎消息
    ↓
进入主循环 (main_loop)
    ↓
等待中断 (HLT)
```

## 设备操作流程

### 读操作流程

```
应用程序调用 device_read
    ↓
发送读命令到控制端口 (0x62)
    ↓
设备处理请求
    ↓
设备触发 0x60 中断
    ↓
中断处理程序响应
    ↓
从数据端口 (0x61) 读取数据
    ↓
返回数据给应用程序
```

### 写操作流程

```
应用程序调用 device_write
    ↓
将数据写入数据端口 (0x61)
    ↓
发送写命令到控制端口 (0x62)
    ↓
设备处理请求
    ↓
设备触发 0x60 中断
    ↓
中断处理程序响应
    ↓
显示完成消息
    ↓
返回控制权给应用程序
```

## 编译和运行

### 编译步骤

**Windows**：
```cmd
build.bat          # 编译完整操作系统
build_test.bat     # 编译测试程序
clean.bat          # 清理编译文件
```

**Linux/Unix**：
```bash
make all           # 编译所有文件
make test          # 编译测试程序
make clean         # 清理编译文件
```

### 运行

**使用 QEMU**：
```cmd
qemu-system-x86_64 -fda os.img   # 运行完整操作系统
qemu-system-x86_64 -fda test.img # 运行测试程序
```

**使用设备模拟器**：
```bash
python3 device_simulator.py      # 运行设备模拟器
```
